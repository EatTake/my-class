# 信息窗与悬浮窗增强功能规格说明

## 一、需求背景

当前游戏界面存在以下问题需要解决：

1. **信息窗宽度问题**：信息窗（`.bottom-bar`）在某些屏幕尺寸下宽度可能超出黑板区域（`.blackboard`）或容器边界，导致布局错乱和横向滚动条出现。

2. **学生属性弹窗溢出问题**：学生属性弹窗（`.student-tooltip`）被限制在 `.desks-area` 容器内，无法在悬浮时正常显示在学生座位区域之外，导致信息被截断。

3. **移动端交互问题**：当前悬浮窗仅支持PC端的鼠标悬停触发，在移动端（无hover事件）无法正常显示学生属性信息。

## 二、修改内容

### 2.1 信息窗宽度约束
- 确保信息窗宽度始终不超过其父容器宽度
- 移除固定宽度百分比，改用 `max-width` 约束
- 确保响应式布局下无横向溢出

### 2.2 学生属性弹窗溢出显示
- 移除弹窗的 `overflow: hidden` 限制
- 调整弹窗定位机制，使其可以显示在容器之外
- 实现点击外部自动隐藏逻辑

### 2.3 移动端触发方案
- **选定方案：点击触发**
- PC端保持鼠标悬停触发
- 移动端点击学生头像显示弹窗，再次点击或点击其他区域隐藏
- 确保两种交互方式互不干扰

## 三、影响范围

### 3.1 受影响的规格
- 响应式布局设计
- 用户交互体验

### 3.2 受影响的代码文件
- `style.css`：弹窗样式、信息窗样式
- `game.js`：学生弹窗显示/隐藏逻辑、移动端事件处理

## 四、详细需求规格

### 4.1 信息窗宽度约束

#### 场景：信息窗在各种屏幕尺寸下正常显示
- **当** 页面加载完成或窗口尺寸发生变化
- **那么** 信息窗宽度应始终 ≤ 黑板区域宽度
- **那么** 不应出现横向滚动条

### 4.2 学生属性弹窗溢出显示

#### 场景：PC端鼠标悬停显示弹窗
- **当** 用户鼠标悬停在学生头像上
- **那么** 弹窗应在学生头像上方/侧方显示
- **那么** 弹窗内容应完整显示，不被容器截断

#### 场景：移动端点击显示弹窗
- **当** 用户点击学生头像（移动端）
- **那么** 弹窗应显示并可超出 `.desks-area` 边界
- **那么** 再次点击其他区域弹窗应隐藏

#### 场景：弹窗超出可视区域
- **当** 弹窗显示位置超出浏览器可视区域
- **那么** 弹窗应自动调整位置确保内容可见

### 4.3 移动端触发逻辑

#### 场景：移动端交互区分
- **当** 检测到触摸设备（touch事件支持）
- **那么** 启用点击触发模式
- **那么** 鼠标悬停事件不再触发弹窗

#### 场景：点击其他区域隐藏
- **当** 弹窗处于显示状态
- **那么** 点击页面任何非弹窗区域应隐藏弹窗

## 五、修改需求

### 5.1 需求：信息窗宽度约束
- 信息窗使用 `max-width` 替代固定宽度百分比
- 在所有响应式断点处保持约束
- 移除任何可能导致溢出的样式

### 5.2 需求：学生属性弹窗溢出显示
- 弹窗定位使用 `fixed` 或脱离文档流的定位方式
- 实现点击外部区域隐藏弹窗功能
- 确保弹窗 `z-index` 足够高不被遮挡

### 5.3 需求：移动端触发方案实现
- 使用点击触发替代悬停触发
- 区分PC端和移动端交互逻辑
- 确保两种模式功能对等
